Communication Protocols:

REST API: Used between client applications and API Gateway for standard HTTP-based communication. Handles most CRUD operations and client-server interactions.

GraphQL: Alternative to REST at the client-API Gateway layer, providing flexible data querying capabilities. Used for complex data queries where clients need to fetch related data in a single request.

gRPC/Proto: Used for high-performance communication between the API Gateway and microservices. All internal service-to-service communication uses gRPC for efficiency and type safety.

Kafka: Used for asynchronous event-driven communication between services. Handles events like property updates, appointment status changes, notifications, etc. Enables decoupled architecture and reliable event delivery.

Data Flow Examples:

  • Property Creation: Client → REST API → API Gateway → gRPC → Property Service → Kafka → Notification Service → Push Notification to User
  • Appointment Scheduling: Client → GraphQL → API Gateway → gRPC → Appointment Service → Kafka → Notification Service + Chat Service
  • User Authentication: Client → REST API → API Gateway → gRPC → User Service → JWT Token → Client
  • Real-time Messaging: Client → WebSocket → API Gateway → gRPC → Chat Service → MongoDB → Kafka → Notification Service
